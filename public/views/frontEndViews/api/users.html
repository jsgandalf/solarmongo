Users

Don't forget to pass in an access_token with every request

  • get: /api/users - gets all users
  • post /api/users - adds a user, pass in a user object
  • get /api/users/:webUserId - gets 1 user, pass in the userid, returns a user object
  • put /api/users/:webUserId - updates a user, pass in the userid and user object
    • user:{
      • "_id": "5302b38828a289d2033f6a61",
      • "email": "sean@gmail.com",
      • "name": "Sean",
      • "role": "Admin",
      • "created": 1292825935491, (milliseconds since 1970/01/01)
      • "updated": 1392874630178, (milliseconds since 1970/01/01)
    • }
  • del /api/users/:webUserId - deletes a user, pass in the userid
